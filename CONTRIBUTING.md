# Contributing to Lemonlime Linux

Thanks for your interest in helping to build Lemonlime Linux!

## Ways to Contribute

- **Report bugs:** File issues with clear steps to reproduce or attach logs/screenshots.
- **Suggest features:** Open a feature request issue; describe your idea and its benefits.
- **Send pull requests:** Fork the repo, branch off from `main`, and submit PRs for bug fixes, features, or documentation.
- **Help with documentation:** Improve guides, tutorials, or translations.
- **Join technical discussions:** Participate in issues, pull requests, or the community chat.

## Development Setup

Please see the `/docs` repo for project-wide setup, build, and tooling instructions for each subsystem.

## Coding Guidelines

- Write clear, readable code.
- Use descriptive commit messages (e.g. `fix(installer): handle empty disk error`).
- Follow conventions and code styles for each repo—refer to repo README or ask via issues.
- Write or update documentation when meaningful.
- Be respectful and helpful in all interactions.

## Pull Request Process

1. Fork the repository and create your branch from `main`.
2. If you've fixed a bug or added code, add relevant tests if possible.
3. Ensure code passes CI (if set up).
4. Open a PR and fill out the template.
5. Reference relevant issues (e.g. “Closes #42”).
6. Collaborate on feedback and iterate!

## Community
